- ⚠️ Loading states: skeleton/spinner on async ops — most pages use spinners; history page uses text-only "loading" with no skeleton — `src/app/[locale]/history/page.tsx:86`
- ⚠️ Empty states: no data, first use — history has empty state; BulkReportView renders empty table rows with no message if `results` is an empty array — `src/components/BulkReportView.tsx:63`
- ⚠️ Form behavior: validation, error messages, tab order — URL/email/OTP validated with inline errors; email capture input lacks `id`/`label` association — `src/app/[locale]/analyze/page.tsx:233`
- ⚠️ Interactive states: hover, focus, active, disabled, loading — buttons have all states; UserNav dropdown missing `aria-expanded`/`aria-haspopup` — `src/components/UserNav.tsx:91`
- ⚠️ Feedback: success/error notifications after actions — most actions provide feedback; sign-out silently closes menu with no confirmation — `src/components/UserNav.tsx:63`

### 2. **API (D2: 58.3%)** 🟡

- ✅ Route definitions match expected CRUD operations — all 14 endpoints cover expected domain patterns
- ⚠️ Status codes: correct codes for success, client error, server error — all routes correct; `/api/email` has no try/catch so DB failure returns unhandled 500 — `src/app/api/email/route.ts:22`
- ⚠️ Input validation: request body/params validated before processing — all routes validate; email validation uses only `includes('@')` — accepts malformed emails — `src/app/api/auth/request-otp/route.ts:18`
- ❌ Error responses: consistent format with message — all routes use `{error:'...'}` format except `/api/email` which crashes without error response — `src/app/api/email/route.ts:22`
- ✅ Pagination: offset/limit/sorting for list endpoints — `/api/history` uses cursor-based pagination with `PAGE_SIZE=20`, `hasMore`, `nextCursor`
- ⚠️ Idempotency: duplicate requests handled safely — auth and settings use upsert correctly; email capture allows duplicate email+slug rows — `src/app/api/email/route.ts:22`

### 3. **Performance (D3: 66.7%)** 🟡

- ✅ N+1 queries: no loops with DB calls inside — `Promise.all` used for parallel AI calls; no DB calls inside loops
- ⚠️ Indexes: queries use indexed columns — `userId` and `ip` indexed separately; no composite index on `(userId, createdAt)` or `(ip, createdAt)` for rate-limit queries — `src/lib/rate-limit.ts:20`
- ⚠️ Payload size: responses don't return unnecessary data — history endpoint parses full multi-KB JSON blob per record just to extract scores — `src/app/api/history/route.ts:36`
- ✅ Async operations: long tasks don't block request — all AI calls and scraping properly async/await with Promise.all parallelism
- ❌ Caching: repeated lookups cached where appropriate — PostHog config queried from DB on every authenticated analysis with no caching — `src/app/api/analyze/route.ts:180`
- ✅ Bundle: no unnecessary imports — Resend SDK dynamically imported; cheerio externalized in `next.config.mjs`

### 4. **Security (D4: 64.3%)** 🟡

- ⚠️ Authentication: all routes behind auth middleware — history, results, settings auth-gated; analyze/bulk/compare open by design; `/api/email` unauthenticated — `src/app/api/email/route.ts:1`
- ✅ Authorization: RBAC checks per endpoint — history and settings scoped to `session.userId`; shared results public by design
- ⚠️ Input sanitization: user input escaped/validated — URLs validated via `new URL()`; emails trimmed/lowercased; scraped content passed to LLM without sanitization (prompt injection risk) — `src/app/api/analyze/route.ts:146`
- ⚠️ No data exposure: API doesn't leak sensitive fields — auth/me returns only id+email; PostHog key masked; shared results include `liveData` from user's private PostHog — `src/app/api/results/[slug]/route.ts:17`
- ❌ Rate limiting: abuse prevention on sensitive endpoints — analyze/bulk/compare rate-limited; **OTP request/verify have NO rate limiting** — brute-force attack possible — `src/app/api/auth/request-otp/route.ts:1`
- ✅ Secrets: no hardcoded credentials, env vars used — all API keys from `process.env`; PostHog keys encrypted at rest
- ✅ SQL injection: parameterized queries — all DB access via Prisma ORM; no raw SQL

### 5. **Data Integrity (D5: 58.3%)** 🟡

- ✅ Migration safety: schema changes are additive/safe — all 3 migrations verified additive-only

## Release Gate

7 P0 findings including unprotected OTP brute-force attack surface, hardcoded encryption fallback key, unconsented PII collection, and missing cookie consent. ## Top Issues

**P0 — Must fix before release:**

1. **OTP endpoint rate limiting missing** — `src/app/api/auth/request-otp/route.ts:1` & `src/app/api/auth/verify-otp/route.ts:1`
   Add IP-based rate limiting (3 requests/15min) and 5-attempt lockout on OTP verify to prevent brute-force account takeover.

2. **Hardcoded encryption fallback key** — `src/lib/encrypt.ts:4`
   Remove fallback; throw unconditionally if `SESSION_SECRET` unset. PostHog API keys are otherwise encrypted with a publicly known key.

3. **Email route crashes on DB failure** — `src/app/api/email/route.ts:22`
   Wrap `prisma.emailCapture.create` in try/catch. Currently unhandled exception returns 500 with stack trace.

4. **averageScore() returns NaN on empty experts** — `src/app/[locale]/analyze/page.tsx:17`
   Add guard: `if (scores.length === 0) return 0`. ScoreGauge renders broken SVG arc when passed NaN.

5. **Email PII collected without GDPR consent** — `src/app/api/email/route.ts:22`
   Add explicit consent checkbox; document lawful basis before collecting email addresses.

6. **No cookie consent banner** — (entire app)
   Implement cookie consent mechanism before any non-essential cookies or tracking pixels are added.

**P1 — Should fix:**

7. **Email capture endpoint unauthenticated** — `src/app/api/email/route.ts:1` — Add auth or CAPTCHA
8. **Form labels not associated with inputs** — `src/components/SignInModal.tsx:117`, `settings/posthog/page.tsx:201` — Add `id`/`htmlFor` pairs
9. **Email capture inputs have no label** — `src/app/[locale]/analyze/page.tsx:233` — Add `aria-label`
10. **UserNav dropdown missing ARIA attributes** — `src/components/UserNav.tsx:91` — Add `aria-expanded`, `aria-haspopup`
11. **No skip navigation link** — (layout) — Add `<a href="#main-content">Skip to content</a>`
12. **No data retention policies** — GDPR Article 5 — Add 90-day cleanup cron
13. **No right-to-erasure endpoint** — GDPR Article 17 — Implement DELETE /api/auth/account
14. **AES-256-CBC should be GCM** — `src/lib/encrypt.ts:12` — Migrate to authenticated encryption
15. **Logger missing request correlation IDs** — `src/lib/logger.ts:1` — Add requestId via AsyncLocalStorage
