## Findings by Dimension

### 1. **UI/UX (D1: 71.4%)**
- ✅ Loading state with cycling expert display every 4s — `src/components/LoadingExpert.tsx`
- ✅ URL auto-prepend `https://` and tab switching (single/compare/bulk) — `src/app/[locale]/page.tsx`
- ✅ Language switcher (EN/VI/DE) with cookie persistence — `src/components/LanguageSwitcher.tsx`
- ✅ Score gauge, expert cards, PostHog guide, comparison banner all rendered
- ⚠️ Empty state missing on analyze result page when `sessionStorage` is empty — `src/app/[locale]/analyze/page.tsx:~15` (redirects, but no graceful fallback message)
- ⚠️ SignInModal: no focus trap — keyboard users can tab outside modal — `src/components/SignInModal.tsx`
- ❌ No disabled state on the analyze button during submission (double-submit risk)

### 2. **API (D2: 83.3%)**
- ✅ All CRUD operations covered; consistent `{ error: "..." }` response format
- ✅ HTTP status codes correct: 400 (validation), 401 (unauth), 429 (rate limit), 500 (server)
- ✅ Input validated before processing on all routes
- ✅ Cursor-based pagination on history endpoint — `src/app/api/history/route.ts:15`
- ⚠️ `/api/results` POST (legacy route) has no authentication guard — anyone can store arbitrary data — `src/app/api/results/route.ts:9`
- ⚠️ No `Retry-After` header on 429 responses — clients cannot back off gracefully — `src/app/api/analyze/route.ts:~95`
- ❌ No idempotency: re-analyzing the same URL creates a new slug and DB row every time

### 3. **Performance (D3: 60.0%)**
- ✅ 6 expert AI calls run in parallel via `Promise.all` — `src/lib/analyze-pipeline.ts:27`
- ✅ Bulk mode runs all URLs in parallel — `src/app/api/bulk/route.ts:74`
- ✅ Cursor pagination prevents unbounded history queries
- ⚠️ No index on `AnalysisResult.ip` or `AnalysisResult.userId` — rate-limit queries scan the table — `prisma/schema.prisma:16`
- ⚠️ History endpoint returns full `data` JSON blob per result before extracting score — `src/app/api/history/route.ts:45` (should store score separately)
- ❌ No caching: same URL analyzed twice runs full scrape + 7 AI calls each time
- ❌ `parseJsonResponse` duplicated in `route.ts` and `analyze-pipeline.ts` — not a perf issue but a maintenance risk

### 4. **Security (D4: 50.0%)**
- ✅ Auth guards on all protected endpoints (history, settings, me)
- ✅ PostHog API key encrypted at rest with AES-256-CBC — `src/lib/encrypt.ts`
- ✅ HttpOnly + SameSite=Lax session cookie — `src/lib/session.ts:47`
- ✅ OTP invalidates prior tokens before issuing a new one — `src/app/api/auth/request-otp/route.ts:19`
- ✅ Prisma parameterized queries — no SQL injection risk
- ❌ **SSRF vulnerability**: `scrapePage()` accepts any URL including `http://localhost`, `http://169.254.169.254` (AWS metadata), internal IPs — `src/lib/scraper.ts:4`
- ❌ **OTP is not cryptographically random**: uses `Math.random()` which is not CSPRNG — `src/app/api/auth/request-otp/route.ts:8`
- ⚠️ Session secret has hardcoded insecure fallback — `src/lib/session.ts:9` (`"dev-secret-must-change-in-production!!"`)
- ⚠️ Session cookie missing `Secure` flag — transmittable over HTTP — `src/lib/session.ts:47`
- ⚠️ `/api/email` has no rate limiting — spam vector for email capture — `src/app/api/email/route.ts`
- ⚠️ `/api/results` POST unprotected — unauthenticated data injection into DB

### 5. **Data Integrity (D5: 58.3%)**
- ✅ Unique constraints: `slug`, `User.email` — `prisma/schema.prisma`
- ✅ FK enforced: `AnalysisResult.userId → User.id`, `PostHogConfig.userId → User.id`

## Top Issues

**P0 — Must fix before release:**

1. **SSRF vulnerability in scraper** — `src/lib/scraper.ts:4`
   `scrapePage()` accepts `http://localhost`, `http://169.254.169.254`, and any private IP. Attackers can probe internal services. Fix: validate URL hostname against a blocklist of private/reserved ranges before fetching.

2. **OTP uses Math.random() (not CSPRNG)** — `src/app/api/auth/request-otp/route.ts:8`
   `Math.floor(100000 + Math.random() * 900000)` is predictable. Replace with `crypto.randomInt(100000, 999999)` (Node.js built-in CSPRNG).

3. **Bulk analysis fails entirely on one bad URL** — `src/app/api/bulk/route.ts:74`
   `Promise.all` rejects on first failure. Use `Promise.allSettled` and return partial results with per-URL error status.

**P1 — Should fix:**

4. **No transaction on OTP verify** — `src/app/api/auth/verify-otp/route.ts:30`
   OTP mark-used and user upsert should be a single `prisma.$transaction([...])` call to prevent race conditions.

5. **Session cookie missing `Secure` flag** — `src/lib/session.ts:47`
   Add `; Secure` to cookie header. Should be conditional on `NODE_ENV === 'production'`.

6. **`/api/results` POST unauthenticated** — `src/app/api/results/route.ts:9`
   Add auth guard or rate-limit; currently allows anyone to inject arbitrary data into DB.

7. **Missing database indexes** — `prisma/schema.prisma:16`
   Add `@@index([userId])` on `AnalysisResult` and `@@index([ip])` for rate-limit query performance.

8. **No max content-length on scraped pages** — `src/lib/scraper.ts:15`
   Add a 5MB response size guard to prevent OOM on large pages.

**P2 — Nice to fix:**

9. **Hardcoded session secret fallback** — `src/lib/session.ts:9`
   Throw at startup if `SESSION_SECRET` is not set in production instead of silently using the dev default.

10. **No health check endpoint** — Missing `/api/health`
    Add `GET /api/health` returning `{ status: "ok" }` for load balancer / uptime monitoring.

11. **Duplicate `parseJsonResponse`** — `src/app/api/analyze/route.ts:20` and `src/lib/analyze-pipeline.ts:9`
    Extract to `src/lib/parse-json.ts` and import in both files.

12. **EmailCapture no slug FK / no uniqueness** — `prisma/schema.prisma:30`
    Add `@@unique([slug, email])` to prevent duplicate captures.

13. **SignInModal lacks ARIA dialog semantics** — `src/components/SignInModal.tsx`
    Add `role="dialog"`, `aria-modal="true"`, `aria-labelledby` and focus trap.

## Recommendation

> **CONDITIONAL**: The core product (AI analysis pipeline, sharing, auth) is functionally solid, but three P0 issues (SSRF, weak OTP randomness, bulk partial-failure) must be resolved before production traffic. The observability and compliance dimensions (D9, D10) are near-zero and represent the largest quality debt — implement structured logging and a basic privacy disclosure as the next sprint priority.
